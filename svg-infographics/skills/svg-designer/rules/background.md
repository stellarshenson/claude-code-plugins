**Background rule card** - rules when preflight declares `--backgrounds N`.

Background textures add depth and thematic flavour without competing for attention. Default posture: faded, layered behind content, never crossed by content groups.

## Layer z-order (MANDATORY)

Background is the FIRST `<g>` inside `<svg>`, after `<style>`. Content groups come after:

```xml
<svg>
  <style>...</style>
  <g id="background">
    <!-- texture elements here -->
  </g>
  <g id="nodes">...</g>
  <g id="connectors">...</g>
  <g id="content">...</g>
</svg>
```

The `<g id="background">` group holds either:
- A solid / gradient backplate (`<rect>` spanning viewBox with `fill=transparent` or theme colour at very low opacity)
- Procedural texture (`<path>` traces, circuit nodes, neural branches, etc.) generated by `svg-infographics background`
- Both

## Opacity range

- Texture stroke / fill: `opacity: 0.06-0.12`. Above 0.15 competes with content; below 0.05 is invisible.
- Solid backplate: `fill-opacity: 0-0.04`. Zero = transparent (matches slide / document bg). Small tint warms the canvas.
- Dark-mode override: slightly higher opacity (0.08-0.15) because dark backgrounds absorb texture more.

## Texture families (`--background-texture` values)

- `circuit` - PCB trace lines with node dots at intersections. Good for tech / data / infrastructure diagrams.
- `neural` - Organic branching (dendrite-like). Good for ML / AI / cognition diagrams.
- `topo` - Contour lines (topographic map style). Good for terrain / landscape / data-flow-as-terrain metaphors.
- `grid` - Engineering grid (coordinate ticks). Good for technical drawings / schematics.
- `celtic` - Interlocking knotwork. Decorative; use for heritage / craft themes only.
- `organic` - Leaf veins / river tributaries. Good for agriculture / biology / natural-system diagrams.
- `none` - Explicitly no texture (solid or transparent backplate only).

`svg-infographics background --family <family> --viewBox "0 0 800 450" --seed <int>` generates the SVG snippet. Seed makes textures reproducible across re-generation.

## Do NOT cross content with texture

Content elements (cards, connectors, text) should not visually sit "underneath" a dark texture trace. If a trace runs through a card's background fill zone, either:
- Increase card `fill-opacity` (0.06 -> 0.10) to occlude the trace
- Offset the card so it sits in a lighter region of the texture
- Regenerate the texture with a different seed

The `svg-infographics background` tool can mask content bounding boxes out of the generated texture (`--mask-bboxes "x,y,w,h x,y,w,h"`). Use this on dense compositions.

## Dark mode (MANDATORY when `--dark-mode required`)

Every texture stroke / fill MUST have a dark-mode override. Standard pattern:

```css
.bg-trace { stroke: #5456f3; stroke-opacity: 0.08; }
@media (prefers-color-scheme: dark) {
  .bg-trace { stroke: #7374ff; stroke-opacity: 0.12; }
}
```

The `background` tool emits this override automatically; hand-crafted backgrounds must add it.

## Group conventions (MANDATORY for check)

The background `<g>` MUST have `id="background"` or `id` ending in `-background` (e.g. `id="circuit-background"`), or `class` containing `"background"`. Without this, `check` counts 0 backgrounds and the declaration fails.
